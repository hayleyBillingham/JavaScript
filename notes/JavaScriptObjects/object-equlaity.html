Object Equality

  • == should be avoided, it is only useful in rare cases
  • == is not type safe
    • "42"== 42
    • 0 == false
    • null == undefined
    • "" == 0
    • [1,2] == "1,2"
    • === Is the most commonly used and should be used in all cases
    • Object.is() is less commonly used. it is like === except for a few mathematical differences
    • === and object.is() are both type-safe
    • === is convientent and concise
    • object.is() is more verbose
    • === NaN is not equal to Nan
    • Object.is() NaN equals NaN
